Analisando o arquivo schema.ts, aqui está a estrutura principal do banco de dados PostgreSQL:

Tabelas Principais:

users (Usuários)
Campos principais: id, email, password, name, userType (client/provider/admin), isActive, isVerified
Armazena todos os usuários do sistema
services (Serviços)
Campos principais: id, providerId, categoryId, name, description, price, duration
Serviços oferecidos pelos prestadores
appointments (Agendamentos)
Campos principais: id, clientId, providerId, serviceId, date, startTime, endTime, status
Registra todos os agendamentos
categories (Categorias)
Campos principais: id, name, description, nicheId, parentId
Categoriza os serviços em grupos
niches (Nichos)
Campos principais: id, name, description
Agrupa categorias em nichos maiores
availability (Disponibilidade)
Campos principais: id, providerId, dayOfWeek, startTime, endTime
Controla horários disponíveis dos prestadores
provider_settings (Configurações do Prestador)
Campos principais: id, providerId, businessName, description, rating
Configurações específicas dos prestadores
reviews (Avaliações)
Campos principais: id, clientId, providerId, appointmentId, rating, comment
Avaliações dos serviços prestados
payment_settings (Configurações de Pagamento)
Campos principais: id, serviceFeePercentage, stripeEnabled, asaasEnabled
Configurações gerais de pagamento
support_tickets (Tickets de Suporte)
Campos principais: id, userId, subject, status, priority
Sistema de suporte ao usuário
Relacionamentos principais:

Usuários (providers) → Serviços
Serviços → Categorias → Nichos
Agendamentos → Usuários (clients e providers)
Avaliações → Agendamentos  > providers
O banco utiliza PostgreSQL com Drizzle ORM para gerenciamento das operações e relacionamentos.
